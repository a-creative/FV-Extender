Version 3.0.8+

Version 3.0.3

This release is a result of grabbing the low hanging fruits due to the health issues mentioned above.

New features

  • In the future you will see this "Updates" page just after the autoupdate of the extension is done and the update is installed.
  • It is now possible to disable the sounds. This way you can use the Extender without your boss wondering why his constantly hears the sound of a cow or a sheep.
  • We've added a "Just help" button which makes the Extender save all requests for later except those related to simply helping your neigbors.
  • So far the Extender has not send return gifts when accepting a "Thank you" gift. There is now an option to enable/disable this. The default is that it will do it.

Fixed bugs

  • When running of out requests the Extender will now ignore the warnings and just continue to the next request and try to proces that.
  • The lastest relaunch of the message center in FV has made some bugs. I've tried to solve these. I won't go into details since the issues wasn't catastrophic.

Known issues

  • FIXED in 3.0.4: The relaunch of the message center introduced a couple of problems I couldn't fix just like that. They are both related to the fact that I suddenly can't proces the user text attached to a request:
    • Requests with user texts are currently not skipped.
    • Sofar writing ex. "Don't resend" would have disabled the return of a gift, but this does not work.

FAQ

  • The new 500 gift limit in the gift box is automatically handled by the Extender but due to FV not returning the correct error messages the Extender has aborted already at 200. The issue is reported to Zynga.
  • Some of you have been wondering why you suddenly get requests with the text "Thank you for sending me such a wonderful gift...". This is not a virus but a result of the relaunch of the message center in FV. The Extender handles these requests like normal "Thank you" gifts.

Version 3.0.2

New features

  • FV Extender can now automatically return a gift when accepting one.
  • Cow sound when the proces is done.
  • Sheep sound when the proces is aborted by FV Extender.
  • "Granted wish", "Thank you" and "Don't resend" requests are now accepted without returning gifts.
  • "New neighbor" requests are skipped for manual processing.
  • Added in 3.0.2: The toolbar promotion screen is now handled as an error screen and FV Extender redirects you to the list of game requests so the processing of requests can continue.

Known issues

  • The auto gift back feature is still experimental but I decided to give it to you anyway. I'm constantly working on making it completely stable. If you experience that the proces stops in the middle of it all please reload the page and click the "Accept all" button again. If it is a constant problem please report it back to me.
  • If you wonder why some of you game requests just don't get accepted it might be because FV Extender skips some types of game requests automatically. Read more about this in the user guide.
  • FV Extender does currently not run in an incognito window. So for now you will have to play with open cards to use it :-)
    Thank you Roy for reporting this bug in the support forum.
  • FIXED in v. 3.0.1. When you reach your gift box limit it will currently abort say something with [BUSHEL LIMIT]. It correctly abort so you can make room in your gift box, but the message could be better. Will fix this ASAP.
  • FIXED in 3.0.2: The "Oh no It looks like all the bits got lost" error screen has so far not been handled correctly. It should be handled correctly now and will return you to the list of game requests so the processing of game requests can continue.
  • FIXED in 3.0.2: Some of you couldnt see the popup when you clicked the "Accept all..." button. The position of the window was simple wrong. Now it's been changed to be in the middle of the screen. That should work for everybody.
